👨‍🎨 开发特别提示：
优先级处理：在函数中，我将 level.hi 放在了最上面。这意味着如果同时触发了两个状态（虽然逻辑上不应发生），系统会优先显示最高级别的报警（红色）。
默认值：如果三个状态都为 0，我设置了默认的灰色或 0 水位，防止界面出现异常空缺。

🚀 最终图纸预览：动态位移流动脚本
这是让液位在预览中真正“流动”起来的核心。请将其放在图纸的 Setup Script (预置脚本) 中：
code
JavaScript
var gv = this;

gv.addScheduleTask({
    interval: 50,
    action: function(data) {
        // 确保该节点拥有液位属性
        if (data.a('waveRange') !== undefined) {
            var hi = data.a('level.hi');
            var low = data.a('level.low');
            var currentOffset = data.a('wareXOffset') || 0;
            
            // 设定流速逻辑
            var speed = 0.05; // 默认流速
            if (hi == 1) {
                speed = 0.12; // 报警时流速变快
            } else if (low == 1) {
                speed = 0.02; // 枯水时流速极慢
            }
            
            // 执行位移累加
            data.a('wareXOffset', currentOffset + speed);
        }
    }
});
👨‍🎨 为什么这样写最专业？
优先级原则：通过 if...else if 结构，我们确保了如果后台数据同时传回 hi:1 和 low:1（虽然物理上不可能，但为了防错），系统会优先显示最严重的红色高位报警。
视觉反馈一致性：正常状态下呈现沉稳的蓝色和均匀的波动。一旦状态切换，颜色、水位高度、起伏剧烈程度以及流速会同步变化。这种“多维度反馈”能让运维人员在不需要阅读文字的情况下，通过眼角余光就能判断水位健康度。
性能友好：函数中使用了 data.a() 获取属性，这在 HT 中渲染效率极高，即便是大屏幕上展示上百个液位计也不会卡顿。
您可以直接将这些函数部署到图纸中进行最终预览。有其他联动逻辑需要优化吗？

💡 专家提示：
数据类型注意：在你的 JSON 定义中，valueType 是 String。所以在写函数时，请使用引号（如 == '1'），否则可能会因为类型不匹配导致逻辑失效。
重叠管理：在 HT 编辑器中，建议将不同状态的水泵放入不同的层级（Layer）或者通过树状列表重命名，方便你准确地选中每一个并绑定 visible 函数。
宝石点位置：我注意到你有一些宝石点是独立的 Shape，记得也要给它们的 background 绑定和文字颜色逻辑一致的函数。
